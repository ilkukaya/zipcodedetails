# ZIPCodeDetails.com

Comprehensive US ZIP code reference platform — location and geographic data for 39,398 ZIP codes.

## Tech Stack

- **Framework**: Astro 5.x (Static Site Generator)
- **Styling**: Tailwind CSS 3.x
- **Hosting**: Netlify (deployed via GitHub Actions)
- **Data**: US Census Bureau (Gazetteer) + USPS Postal Data (public domain)
- **Maps**: Leaflet.js (lazy loaded)
- **Analytics**: Plausible (cookie-free)

## Project Structure

```
zipcodedetails/
├── data/               # ZIP code JSON data files (generated by build_data.py)
│   ├── zips/           # Individual ZIP code files (one per ZIP)
│   ├── state_index.json
│   ├── city_index.json
│   ├── county_index.json
│   └── popular_zips.json
├── scripts/            # Python data pipeline
│   └── build_data.py   # Single script — builds all data from pseo_zipcodes_full.json
├── src/
│   ├── components/     # Astro components
│   ├── layouts/        # Page layouts (Base.astro)
│   ├── lib/            # TypeScript utilities (formatters, seo, geo)
│   └── pages/          # Route pages
│       ├── zip/[code].astro      # ZIP detail pages (~39K)
│       ├── city/[slug].astro     # City pages (~29K)
│       ├── county/[slug].astro   # County pages (~3.2K)
│       ├── state/[state].astro   # State pages (59)
│       └── sitemap-*.xml.ts      # Sitemap fragments
└── public/             # Static assets (robots.txt, llms.txt, og-default.png, …)
```

## Getting Started

### Prerequisites

- Node.js 20+
- Python 3.11+ (for data pipeline only)

### Development

```bash
npm install
npm run dev
```

### Data Pipeline

Requires `data/raw/pseo_zipcodes_full.json` (USPS + Census enriched data).

```bash
pip install -r requirements.txt
python scripts/build_data.py
```

This generates all files under `data/` (zips/, state_index.json, city_index.json, county_index.json, popular_zips.json).

### Build

```bash
npm run build
```

## Environment Variables

| Variable   | Default                                   | Purpose                            |
|------------|-------------------------------------------|------------------------------------|
| `SITE_URL` | `https://zipcodedetails.netlify.app`      | Canonical URL base used by Astro (`site`), Plausible domain, and all schema URLs |

When switching to a custom domain:
1. Netlify dashboard → Environment variables → set `SITE_URL = https://zipcodedetails.com`
2. Trigger a new deploy (empty commit or manual trigger in Netlify)
3. All canonical URLs, schema URLs, sitemap URLs, and analytics domain update automatically

## Deployment

All deploys go through **GitHub Actions** → Netlify. Netlify is configured with `ignore = "exit 0"` so it never auto-builds (it would deploy without data). The GitHub Actions workflow builds the site with data and uploads the `dist/` folder to Netlify.

## Data Sources

- [Census Bureau Gazetteer Files](https://www.census.gov/geographies/reference-files/time-series/geo/gazetteer-files.html)
- USPS Postal Data (public domain, sourced via uszipcode-project)
